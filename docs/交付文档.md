## 部署指南

### 后端部署
```bash
# 1. 配置环境变量
cp backend/.env.example backend/.env
# 编辑 backend/.env，填入阿里云凭证

# 2. 启动后端
cd backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000

# 3. Nginx 代理（如需）
# 参考 docs/nginx_sse_config.md 配置 Nginx
```

### Android 打包
```bash
# Debug 构建（已验证）
flutter build apk --debug

# Release 打包（需要签名配置）
flutter build apk --release
```

```bash
cd ios
pod install --repo-update
flutter build ios --release
```

## 技术选型说明

### 为什么选择 Whisper-tiny + Qwen-VL-1.8B？
- Whisper-tiny-int8 TFLite：仅 31MB，适合移动端，int8 量化保证精度
- Qwen2-VL-1.8B Q4_K_M GGUF：约 1.1GB，支持视觉理解，端侧运行零 API 费用

### 为什么选择 FastAPI 而不是 Django/Flask？
- Starlette 原生支持 SSE，性能优于 Flask
- 与 FastAPI 社区生态成熟，阿里云 SDK 支持好

### 为什么需要 Nginx 反向代理？
- 解决 SSE 长连接与 WebServer 超时冲突
- 支持 GZip 压缩节省 SSE 文本带宽
- 统一 HTTPS 出口，隐藏后端真实地址

## 已知限制

1. **端侧模型需手动下载**：模型文件需从 HuggingFace 下载后放入 `assets/` 目录
2. **iOS 打包需要 macOS**：Flutter iOS 打包必须在 macOS 上运行 Xcode
3. **阿里云密钥需手动配置**：首次运行需要填写 `backend/.env` 中的真实凭证
4. **SSE 连接数限制**：Nginx 默认 1 worker 512 连接，大并发需调优
