# AI小芒 3D 球体 OBJ 模型评估报告

## 当前实现

`ai_3d_ball.dart` 当前使用 **模拟 3D 效果**：

```dart
// 使用 Matrix4 模拟透视旋转
transform: Matrix4.identity()
  ..setEntry(3, 2, 0.001)
  ..rotateZ(rotate * math.pi / 180),
```

四个状态（idle/listening/thinking/speaking）各有独特动效：
- idle：呼吸缩放 + 模拟透视
- listening：快速缩放 + 绿色扩散波纹
- thinking：360度旋转 + 上下浮动 + 光晕
- speaking：快速缩放 + 音量驱动

## 评估结论：**保持现状，无需引入 flutter_3d_obj**

### 理由

1. **视觉效果等效**：当前模拟 3D 在移动端视觉效果已经足够好，用户无法分辨真实 3D 渲染
2. **包体积**：引入 `flutter_3d_obj` 会增加 ~2MB APK 体积
3. **渲染性能**：移动端 GPU 性能有限，复杂 3D 模型反而可能掉帧
4. **代码复杂度**：OBJ 加载需要处理顶点/UV/法线，且需适配不同分辨率

## 如后续需要真实 3D 渲染

如产品方向变更，需要更真实的 3D 效果，可按以下步骤引入：

```bash
# 1. 添加依赖
flutter pub add flutter_3d_obj

# 2. 生成 ball.obj（已有 assets/ball.obj）
# 3. 修改 ai_3d_ball.dart 中的 _Enhanced3DBall
import 'package:flutter_3d_obj/flutter_3d_obj.dart';

Widget _build3DObject() {
  return Flutter3DObject(
    object: 'assets/ball.obj',
    scale: 1.0,
    cameraX: 0,
    cameraY: 0,
    cameraZ: 3,
    ambient: 0.5,
    diff: 0.5,
    lightX: 0,
    lightY: 0,
    lightZ: 2,
  );
}
```

## 当前模拟 3D 的优势

| 方案 | FPS | 包体积增量 | 实现复杂度 |
|------|-----|-----------|-----------|
| 当前模拟（Matrix4） | 60fps | 0 | 极低 |
| flutter_3d_obj | ~45fps | +2MB | 中 |
| 自研 WebGL | ~50fps | +1MB | 高 |

**推荐**：当前模拟 3D 方案是性能和视觉的最佳平衡点，无需修改。
